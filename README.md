\# 🌲 熊出没·森林保卫战 (Boonie Bears Survivor)



基于 C++ 与 EasyX 图形库开发的一款 \*\*类幸存者 (Survivor-like) 肉鸽生存割草游戏\*\*。本项目原生支持单人闯关与本地双人同屏合作，融入了经典的 Roguelike 随机升级成长系统。



\## ✨ 核心亮点 (Features)



\*   \*\*👥 双人同屏合作\*\*：支持本地双人游玩，独立物理键盘控制与独立生命槽，社交属性拉满。

\*   \*\*⚔️ 差异化流派设计\*\*：摒弃换皮，3 名经典 IP 角色拥有完全独立的武器弹道（极坐标环绕、正弦波脉冲、直线对象池）与主动技能。

\*   \*\*🧬 肉鸽成长机制\*\*：磁吸经验球拾取，升级触发“随机三选一”强化面板，自由构筑战斗流派（属性增益、减 CD、全屏落雷大招等）。

\*   \*\*⚙️ 动态难度调度\*\*：随着生存时间推移（森林 -> 机械危机 -> Boss降临），敌人血量、移速与 AI 机制动态攀升。

\*   \*\*💥 硬核视听反馈\*\*：双缓冲技术 0 闪烁渲染。包含 AABB 极限擦弹判定、受击霸体/击退、全屏伤害飘字、动态屏幕震动及 3.5 秒阵亡慢动作特写。

\## 🎮 操作说明 (Controls)



\### 界面交互

\*   \*\*鼠标左键\*\*：点击 UI 按钮、选择升级强化卡片。

\*   \*\*鼠标滚轮\*\*：在暂停界面中滚动调节游戏全局音量。

\*   \*\*`TAB` 键\*\*：在主菜单切换 1P 角色。

\*   \*\*`Q` 键\*\*：在主菜单切换 2P 角色（仅双人模式有效）。

\*   \*\*`Y` 键\*\*：在主菜单一键切换【单机割草】与【双人同屏】模式。



\### 战斗操作

| 动作 | 1P (玩家一) | 2P (玩家二) |

| :--- | :--- | :--- |

| \*\*移动\*\* | `W` `A` `S` `D` | `↑` `↓` `←` `→` (方向键) |

| \*\*释放技能\*\* | `Space` (空格键) | `Numpad 0` (数字小键盘 0) |



\### 系统热键

\*   \*\*`P`\*\*：暂停 / 恢复游戏

\*   \*\*`R`\*\*：游戏结束后快速重开 (Restart)

\*   \*\*`M`\*\*：返回主菜单 (Menu)

\*   \*\*`ESC`\*\*：直接退出程序

\## 🏗️ 核心架构与设计模式 (Architecture)



本项目在 C++ 底层实践了大量经典的面向对象（OOP）编程思想：



\*   \*\*策略模式 (Strategy Pattern)\*\*：将角色本体与攻击逻辑解耦，通过 `Weapon` 抽象基类向下派生出 `RockWeapon`、`GunWeapon` 等，实现武器热插拔。

\*   \*\*工厂模式 (Factory Pattern)\*\*：通过 `CreatePlayer` 接口封装对象的实例化过程，主程序无需关注角色初始化的底层装配细节。

\*   \*\*组件化设计 (Component-Based)\*\*：将复杂的“轨道落雷”大招抽离为独立的 `OrbitalStrikeSkill` 组件，内置时间轮询、预判锁敌与多阶段特效管线。

\*   \*\*对象池技术 (Object Pool)\*\*：在光头强的弹幕生成与废弃经验球的内存回收中，采用 Swap-Pop 机制实现 O(1) 复杂度的垃圾回收，保证海量实体同屏 144 帧不卡顿。
